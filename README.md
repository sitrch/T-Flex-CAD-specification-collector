# T-FLEX CAD Specification Collector

Инструмент для сбора и экспорта спецификаций из T-FLEX CAD в Excel.

## Предыстория

Встроенный механизм формирования спецификаций в T-Flex начал подводить на больших проектах. Пока чертежей мало — всё летает, но когда сборка разрастается, T-Flex начинает терять целые ветки: какие-то фрагменты просто выпадают из спецификации. Скорее всего, дело в кривом кэшировании — я не брался разбираться досконально.

Наверное, не самый удачный выбор — использовать CAD для машиностроения в строительных задачах такого масштаба. Но возиться с этой проблемой было интересно, и я ни о чём не жалею: опыт есть опыт, и неважно, положительный он вышел или отрицательный.

## Описание

Приложение подключается к активному документу T-FLEX CAD через COM-интерфейс (TFlexAPI), открывает файлы плоскостей изделия (grb) и выполняет рекурсивный обход структуры изделия (ProductStructure). Для каждого этажа программно устанавливаются переменные T-FLEX (`$Плоскость`, `Этаж`, `$Этажей`), обновляется структура изделия, затем собираются все строки спецификации, отмеченные флагом «Включать в спецификацию». 

Сбор выполняется в два этапа: сначала извлекаются «родные» строки текущего документа, затем обходятся вложенные фрагменты, прошедшие фильтр `IncludeInDoc` — это позволяет корректно собирать многоуровневые сборки, где T-FLEX теряет ветви.

Конфигурация изделий, плоскостей, этажей и захваток задаётся во внешних Excel-файлах (`Files/Захватки.xlsx`, `Files/Этажи.xlsx`). Результат выгружается в два выходных Excel-файла: сводная спецификация с группировкой по разделам и детализация по этапам монтажа.

## Возможности

- **Рекурсивный обход структуры изделия** T-FLEX CAD по схеме спецификации `m2spec` — сбор всех строк, прошедших фильтр `IncludeInDoc`
- **Двухпроходный сбор**: сначала родные строки документа, затем строки из вложенных фрагментов (решает проблему пропуска ветвей на больших сборках)
- **Автоматическая установка переменных** `$Плоскость`, `Этаж`, `$Этажей` и принудительное обновление структуры изделия для каждого этажа
- **Обход нескольких плоскостей изделия** (Изделия 1–4) с независимой конфигурацией этажей и файлов
- **Типизация этажей** — маппинг физических этажей на типовые через внешний файл `Этажи.xlsx`; сбор выполняется по уникальным типовым этажам, количество одинаковых этажей подставляется в переменную `$Этажей`
- **Динамические расчётные столбцы** — вычисление количества вхождений типового этажа в заданных диапазонах (например, `Заказ_2-57`, `Количество_2-57`) с автоподстановкой в Excel
- **Умный расчёт материалов** — для разделов материалов количество умножается на погонный метраж (`Длина, м`), для остальных — на штучное количество
- **Экспорт в Excel с группировкой по разделам** на отдельные листы: Детали, Термомосты, Заполнения, Комплектующие, Материалы, Раскрой, Проверка, Другое (сводный лист «Спецификация»)
- **Детализация по этапам монтажа** — отдельный файл `Спецификация_Этапы.xlsx` с раскрытием каждого физического этажа из типового и привязкой к захваткам (этапам) из файла `Захватки.xlsx`
- **Отслеживание стоек** — чтение переменной `Стойка` из фрагментов стоек и добавление номера стойки в выходные данные
- **Разделение «Ригель / Стойка»** — автоматическое определение размещения записи по цепочке родительских фрагментов
- **Блокировка выходных файлов** — монопольная блокировка Excel-файлов после начала работы программы для предотвращения случайных изменений
- **Обработка заблокированных файлов** — диалог с опциями «Повторить», «Сохранить как» или «Отмена» при попытке записи в занятый файл
- **Валидация входных файлов** — проверка наличия и читаемости `Захватки.xlsx` и `Этажи.xlsx` перед стартом
- **Логирование с дедупликацией** — повторяющиеся сообщения группируются с подсчётом количества, сохраняются в `collector.log1`

## Структура проекта

- `FragmentSpecificationCollector.cs` — основной класс сбора данных
- `SpecData.cs` — конфигурация изделий и плоскостей
- `SpecificationRow.cs` — модель строки спецификации
- `ExcelExporter.cs` — экспорт в Excel через MiniExcel
- `PhaseReader.cs` — чтение данных по этажам из файла
- `FloorMatcher.cs` — сопоставление и группировка этажей
- `LogService.cs` — сервис логирования
- `ModelObject.cs` — модель объекта изделия

## Используемые файлы

- `Files/Захватки.xlsx` — конфигурация захваток (секций сборки)
- `Files/Этажи.xlsx` — конфигурация этажей и их привязка к плоскостям изделия
- `Data.xml` — сохраняемая история изменений этажей (формируется автоматически)

## Порядок столбцов в Excel

Плоскость, Этаж, Раздел, Спецификация, Источник, Место установки, Артикул, Артикул базовый, Наименование, Длина, Длина, м, Значение всего, Количество всего, Заполнения тип, Заполнения ширина, Заполнения высота, Заполнения толщина, Заполнения площадь, Этажей
