# Архитектура проекта SpecCollector

## Общее описание
Инструмент для автоматизации сбора спецификаций из T-FLEX CAD и экспорта их в Excel.

## Основные модули и классы

### 1. Сбор данных (FragmentSpecificationCollector)
**Файл:** `SpecCollector/FragmentSpecificationCollector.cs`
**Назначение:** Главный класс для автоматизации сбора спецификаций из T-FLEX CAD.

**Основные методы:**
- `GenerateSpec()` — точка входа: получает активный документ, обрабатывает Изделия 1-4, экспортирует результат.
- `CollectSpecData(SpecItem, List<Dictionary>)` — обработка конкретной плоскости: открывает .grb файл, перебирает этажи, обновляет переменные `$Плоскость` и `Этаж`.
- `CollectFromDocument(Document, string, int, int)` — рекурсивный обход структуры изделия, вызов `ProcessDocument`.
- `ExportToMultipleSheets(string, List<Dictionary>)` — экспорт в Excel с разбивкой на листы по разделам.
- `UpdateProductStructure(Document)` — обновление структуры "m2spec" перед сбором данных.

**Алгоритм работы:**
1. Для каждого изделия (Изделия1-4) открывается файл чертежа.
2. Для каждого этажа устанавливаются переменные документа и обновляется структура.
3. Рекурсивно обходится структура изделия через `ProcessDocument` за один проход по BOM-строкам:
    - **Фильтр ветвей:** строки с `IncludeInDoc == false` пропускаются (отсечение всей ветви).
    - **Родная строка** (`SourceRowElementUID == Guid.Empty`) → добавляется в `_results`.
    - **Заимствованная строка** (`SourceRowElementUID != Guid.Empty`) → рекурсия в дочерний фрагмент через `SourceFragmentFirstLevel`.
4. Данные группируются по разделам и сохраняются в `Спецификации.xlsx` через MiniExcel.

### 2. Конфигурация (SpecData)
**Файл:** `SpecCollector/SpecData.cs`
**Назначение:** Статический класс-конфигуратор с настройками изделий и правилами группировки.

**Структура данных:**
- `SpecItem` — описание изделия (Плоскость, FileName, массив Этажи).
- `SpecItemQuantity` — описание этажа (Этаж, Этажей).

**Настройки:**
- `плоскостьVarName = "$Плоскость"`, `ЭтажVarName = "Этаж"` — имена переменных T-FLEX.
- `плоскость1-4` — строковые идентификаторы плоскостей: "(5-1)-(5-6)", "(5-6)-(5-1)", "(5-А)-(5-Л)", "(5-Л)-(5-А)".
- `BOMSections` — словарь маппинга разделов в листы Excel (например, "Детали\\Ригели" → "Детали").

**Настроенные изделия:**
- `Изделия1`: Плоскость 1, файл "Этаж12_Чертежи.grb", этажи 2, 3, 4, 19, 20, 40, 57.
- `Изделия2`: Плоскость 2, файл "Этаж12_Чертежи.grb", этажи 2, 3, 4, 19, 20, 57.
- `Изделия3`: Плоскость 3, файл "Изделие3_Чертежи.grb", этажи 2, 3, 19, 29, 30, 39, 57.
- `Изделия4`: Плоскость 4, файл "Изделие4_Чертежи.grb", этажи 2, 3, 19, 29, 30, 39, 57.

### 3. Модель данных (SpecificationRow)
**Файл:** `SpecCollector/SpecificationRow.cs`
**Назначение:** Представляет одну запись спецификации с поддержкой экспорта в Excel.

**Основные свойства:**
- Контекст: `Плоскость`, `Этаж`, `Этажей`, `Источник`.
- Спецификация: `Артикул`, `АртикулБазовый`, `Наименование`, `Раздел`, `Спецификация`, `МестоУстановки`.
- Геометрия: `Длина`, `ДлинаМ`, `ЗначениеВсего`, `КоличествоВсего`.
- Заполнения: `ЗаполненияТип`, `ЗаполненияШирина`, `ЗаполненияВысота`, `ЗаполненияТолщина`, `ЗаполненияПлощадь`.

**Методы:**
- `ToExcelRow()` — преобразует объект в `Dictionary<string, object>` для MiniExcel (использует `DBNull.Value` для null).
- `GetColumnNames()` (static) — возвращает список из 19 имен столбцов в правильном порядке.

### 4. Экспорт (ExcelExporter)
**Файл:** `SpecCollector/ExcelExporter.cs` и `ExcelExporter_part1.cs`
**Назначение:** Запись собранных данных в многостраничный Excel-файл.

**Основные методы:**
- `Export(List<SpecificationRow>)` — главный метод экспорта с обработкой блокировок файла.
- `IsFileLocked(string)` — проверяет, не открыт ли файл в Excel.

**Алгоритм экспорта:**
1. Механизм повторных попыток (до 3 раз) при блокировке файла.
2. При блокировке показывает `FileLockedDialog` с вариантами: "Повторить", "Сохранить как", "Отмена".
3. Создает словарь листов: главный "Спецификация" + листы по разделам.
4. Фильтрует строки (включает только со `Спецификация == "1"`) для листов разделов.
5. Использует `MiniExcel.SaveAs` с `FastMode = true`.

**Порядок листов:** Спецификация → Детали → Термомосты → Заполнения → Комплектующие → Материалы → Раскрой → Проверка → Другое.

### 5. Вспомогательные классы

**SpecDebugService** (`SpecDebugService.cs`)
- Сервис логирования процесса сбора.
- Метод `Show()` открывает окно отладки, `Log(string)` добавляет запись.

**ExcelMiniDataBlock** (`ExcelMiniDataBlock.cs`)
- Вспомогательная структура для работы с данными Excel.

**FileLockedDialog** (`FileLockedDialog.cs`)
- Диалоговое окно, предлагающее действия при заблокированном файле Excel.

**DebugProgressForm** (`DebugProgressForm.cs`)
- Форма отображения прогресса сбора спецификаций.

## Поток данных
1. `FragmentSpecificationCollector` запрашивает данные из `SpecData`.
2. Обход структуры изделия (плоскости 1-4).
3. Формирование списка `List<SpecificationRow>`.
4. Передача списка в `ExcelExporter` для сохранения в `.xlsx`.

## Зависимости
- **T-FLEX CAD API (TFlexAPI.dll):** Доступ к структуре изделия.
- **MiniExcel:** Легковесный экспорт в Excel.
