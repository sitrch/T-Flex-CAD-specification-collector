# SpecData

## Назначение
Статический класс, содержащий конфигурационные данные для сбора спецификаций. Определяет плоскости, файлы изделий, этажи и правила группировки разделов спецификации.

## Структура данных

### Константы плоскостей
- `плоскость1 = "(5-1)-(5-6)"`
- `плоскость2 = "(5-6)-(5-1)"`
- `плоскость3 = "(5-А)-(5-Л)"`
- `плоскость4 = "(5-Л)-(5-А)"`

### Имена переменных T-FLEX
- `плоскостьVarName = "$Плоскость"`
- `ЭтажVarName = "Этаж"`
- `ЭтажейVarName = "$Этажей"`

### Структура `SpecItem`
Описывает одно изделие (плоскость):
- `Плоскость` — строковый идентификатор плоскости.
- `FileName` — имя файла чертежа (например, "Этаж12_Чертежи.grb").
- `Этажи` — массив структур `SpecItemQuantity`.

### Структура `SpecItemQuantity`
- `Этаж` — номер этажа.
- `Этажей` — количество однотипных этажей.

### Маппинг разделов `BOMSections`
Словарь для группировки разделов спецификации в листы Excel:
- "Детали\\Крышка ригеля" → "Детали"
- "Комплектующие" → "Комплектующие"
- "Уплотнители" → "Материалы"
- "Термомосты" → "Термомосты"
- "Раскрой" → "Раскрой"
- "Заполнения" → "Заполнения"
- "Проверка\\Набор ригеля" → "Проверка"
- И другие...

## Настроенные изделия
- `Изделия1` — Плоскость 1, файл "Этаж12_Чертежи.grb", этажи: 2, 3, 4, 19, 20, 40, 57.
- `Изделия2` — Плоскость 2, файл "Этаж12_Чертежи.grb", этажи: 2, 3, 4, 19, 20, 57.
- `Изделия3` — Плоскость 3, файл "Изделие3_Чертежи.grb", этажи: 2, 3, 19, 29, 30, 39, 57.
- `Изделия4` — Плоскость 4, файл "Изделие4_Чертежи.grb", этажи: 2, 3, 19, 29, 30, 39, 57.

## Использование
Класс используется `FragmentSpecificationCollector` для:
1. Получения списка изделий для обработки.
2. Определения маппинга разделов при экспорте в Excel.
3. Доступа к переменным T-FLEX CAD.
