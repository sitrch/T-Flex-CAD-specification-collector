# FragmentSpecificationCollector

## Назначение
Основной класс для сбора спецификаций из документов T-FLEX CAD. Автоматизирует обход структуры изделия, сбор данных из 4 плоскостей (Изделия 1-4) и экспорт в Excel.

## Зависимости
- `TFlex` — API T-FLEX CAD для работы с документами и структурой изделия.
- `SpecData` — конфигурация изделий, плоскостей и правил группировки.
- `SpecificationRow` — модель строки спецификации.
- `ExcelExporter` (встроенный метод `ExportToMultipleSheets`) — экспорт данных.
- Логирование в файлы `.log1` (например, `collector.log1`) через `System.IO.File.AppendAllText`.

## Публичные поля
- `OnlyBOMGenerate` (bool) — если true, генерирует только BOM без полной обработки.
- `IncludeAllRows` (bool) — режим отладки. Если true — отключает фильтрацию по `IncludeInDoc` на ЭТАПЕ 1, позволяя собирать все родные строки (включая `Спецификация == "0"`).

## Подробный алгоритм работы методов

### 1. `GenerateSpec()`
**Назначение:** Главный метод запуска процесса сбора спецификаций.
**Алгоритм:**
1. Получает активный документ T-FLEX CAD.
2. Инициализирует логирование в `collector.log1`.
3. Определяет путь для сохранения файла `Спецификации.xlsx`.
4. Последовательно вызывает `CollectSpecData` для каждого изделия (Изделия1-4).
5. Вызывает `ExportToMultipleSheets` для сохранения в Excel.

### 2. `CollectSpecData(SpecItem изделие, List<Dictionary<string, object>> allData)`
**Назначение:** Сбор данных для конкретного изделия (плоскости).
**Алгоритм:**
1. Формирует путь к файлу документа плоскости.
2. Открывает документ через `TFlex.Application.OpenDocument`.
3. Для каждого этажа в `изделие.Этажи`:
    - Устанавливает переменные `$Плоскость` и `Этаж` в документе.
    - Обновляет структуру изделия через `UpdateProductStructure`.
    - Вызывает `CollectFromDocument` для сбора данных.
    - Преобразует результаты в формат Excel (`ToExcelRow`) и добавляет в `allData`.
4. Закрывает документ после обработки.

### 3. `CollectFromDocument(Document doc, string плоскость, int этаж, int этажей)`
**Назначение:** Рекурсивный сбор данных из структуры документа.
**Алгоритм:**
1. Сохраняет контекст (плоскость, этаж, количество этажей).
2. Вызывает `ProcessDocument` для рекурсивного обхода структуры изделия.

### 4. `ProcessDocument(Document doc, int level, List<string> parentChain = null)`
**Назначение:** Рекурсивный обход документа в два этапа (согласно системному промпту раздел 8).
**Цепочка родительских объектов:** В лог выводится цепочка `parentChain` (через `→`), показывающая путь от корневого документа до текущего. Для получения имени объекта используется `GetDocumentDisplayName(doc)` (переменная `$Наименование` или имя файла).
**Алгоритм:**

**ЭТАП 1: Проход по BOM-строкам (один проход)**
1. Получает структуру изделия "m2spec" из документа. Если отсутствует — выход.
2. Получает схему и все RowElements.
3. Для каждой строки в allRows:
    - **Фильтрация ветвей:** Если `!IncludeAllRows && !IsInSpecification(row)` → строка не включена в спецификацию, пропускаем (счётчик `skippedSpec++`).
    - **Свои строки:** Если `SourceRowElementUID == Guid.Empty` → родная строка → добавляется в `_results` через `ExtractRowFromRowElement`.
    - **Заимствованные строки:** Если `SourceRowElementUID != Guid.Empty` → просто увеличиваем `borrowedCount` (обработка через ЭТАП 2).

**ЭТАП 2: Проход по фрагментам документа (один проход)**
1. Получаем все фрагменты документа через `doc.GetFragments()`.
2. **Фильтрация:** для каждого фрагмента проверяем, есть ли он в коллекции `approvedFragments` (сравнение по ссылке, через `ReferenceComparer`). Если фрагмент НЕ в одобренных — вся ветвь отсекается (`Fragment BRANCH CUT`).
3. Для каждого одобренного фрагмента:
    - Пропускаем, если `frag.FilePath` пустой.
    - Регенерируем фрагмент `frag.Regenerate(true)`.
    - Получаем документ фрагмента `frag.GetFragmentDocument(true)`.
    - Выполняем рекурсивный вызов `ProcessDocument(subDoc, level + 1)`.

**Связь ЭТАП 1 ↔ ЭТАП 2:** В ЭТАПЕ 1 для BOM-строк, прошедших `IsInSpecification`, ссылка `row.SourceFragmentFirstLevel` добавляется в `HashSet<object> approvedFragments`. В ЭТАПЕ 2 фрагмент из `GetFragments()` проверяется по этому набору (reference equality).

**Ключевое разделение:**
- **Логика обработки строк:** Фильтрация по `IncludeInDoc` и добавление в результат только родных строк (ЭТАП 1).
- **Логика обхода фрагментов:** Навигация по дереву фрагментов и рекурсивный вызов `ProcessDocument` (ЭТАП 2).

**Критерии отбора:**
- В результат ДОЛЖНЫ попадать только строки, родные для фрагмента (`SourceRowElementUID == Guid.Empty`).
- В результат НЕ должны попадать строки, заимствованные из дочерних элементов.
- Ветви с корнем, у которого `IncludeInDoc == false`, полностью исключаются из обхода (кроме режима `IncludeAllRows = true`).

**Диагностика:** Логирует этапы работы в `collector.log1`. Логи включают:
- Цепочку родительских объектов (`chain`) для каждой строки (OWN ROW, SKIPPED, BORROWED)
- `IncludeInDoc` и `IncludeInAssembly` — два способа определения флага включения в спецификацию (через RowElement)
- Для каждого фрагмента: тип фрагмента (`Fragment type`), свойство `NeverIncludeInPS` (через рефлексию)
- Для каждого документа фрагмента: переменные `$Артикул` и `$Спецификация`
- В `ExtractRowFromRowElement`: переменная `Спецификация` из документа (числовая — читается через `RealValue`), `IncludeInAssembly` из RowElement
- По умолчанию логируются только строки с артикулом, содержащим "F50.0421" (можно переопределить через `LogFilterArticul`)

### 4.1. `GetDocumentDisplayName(Document doc)` → `string`
**Назначение:** Получает отображаемое имя документа для логов.
**Алгоритм:**
1. Пытается получить переменную `$Наименование` через `doc.FindVariable("$Наименование")`.
2. Если переменная существует и не пустая — возвращает её значение (`var.TextValue`).
3. Если переменной нет — возвращает имя файла документа (`Path.GetFileName(doc.FileName ?? "Unknown")`).

### 5. `IsInSpecification(RowElement)` → `bool`
**Назначение:** Определяет, включён ли фрагмент в спецификацию по полю `IncludeInDoc`.
**Алгоритм:**
1. Получает значение `rowElement.IncludeInDoc?.Value`.
2. Возвращает `true`, если значение существует и равно `true`.

**Примечание:** Проверка `IsInSpecification` может быть отключена установкой флага `IncludeAllRows = true` (режим отладки).

### 6. `ExtractRowFromRowElement(RowElement, Scheme, Document)` → `SpecificationRow`
**Назначение:** Извлечение данных одной записи спецификации.
**Фильтрация:**
- Только строки, прошедшие проверку в `ProcessDocument`.

**Диагностика:** Логирует значения `IncludeInDoc` и итоговой колонки `Спецификация` для строк, попадающих под фильтр (по умолчанию "F50.0421").

### 7. `ExportToMultipleSheets(string filePath, List<Dictionary<string, object>> allData)`
**Назначение:** Экспорт данных в Excel с разбивкой на листы по разделам.
**Алгоритм:**
1. Создает словарь листов `sheets`.
2. Добавляет полный набор данных на лист "Спецификация" (ВСЕ данные, включая `Спецификация == "0"`).
3. Группирует данные по разделам используя `SpecData.BOMSections`:
    - **Фильтрация:** Пропускает строки, где `Спецификация != "1"` (только для листов разделов, не для главного листа "Спецификация").
4. Создает отдельные листы: "Детали", "Термомосты", "Заполнения", "Комплектующие", "Материалы", "Раскрой", "Проверка", "Другое".
5. Использует `MiniExcel.SaveAs` с конфигурацией `FastMode = true`.

### 8. `UpdateProductStructure(Document doc)`
**Назначение:** Обновление структуры изделия перед сбором данных.
**Алгоритм:**
1. Получает все структуры изделия документа.
2. Находит структуру с именем "m2spec".
3. Вызывает `UpdateStructure()` для актуализации данных.

## Связи с другими классами
- Использует `SpecData` для получения конфигурации изделий и маппинга разделов.
- Формирует объекты `SpecificationRow` при обходе структуры.
- Экспортирует данные через встроенный механизм `MiniExcel`.
