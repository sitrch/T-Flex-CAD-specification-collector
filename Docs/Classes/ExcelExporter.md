# ExcelExporter

## Назначение
Класс для экспорта собранных записей спецификации в файл Excel. Использует библиотеку `MiniExcel` для быстрого создания многостраничных документов.

## Зависимости
- `MiniExcelLibs` — основная библиотека для работы с Excel.
- `MiniExcelLibs.OpenXml` — конфигурация для формата OpenXML.
- `SpecData` — для получения маппинга разделов `BOMSections`.
- `SpecificationRow` — для получения имен столбцов через `GetColumnNames()`.

## Подробный алгоритм работы методов

### 1. `ExcelExporter(string filePath)`
**Назначение:** Конструктор, инициализирует путь к файлу экспорта.

### 2. `IsFileLocked(string filePath)`
**Назначение:** Проверяет, занят ли файл другим приложением.
**Алгоритм:**
1. Если файл не существует, возвращает `false`.
2. Пытается открыть файл с `FileAccess.ReadWrite` и `FileShare.None`.
3. Если успешно — файл не заблокирован.
4. При `IOException` — файл занят другим процессом (например, открыт в Excel).

### 3. `Export(List<SpecificationRow> rows)`
**Назначение:** Основной метод экспорта списка записей в Excel.
**Алгоритм:**
1. Реализует механизм повторных попыток (до 3 раз) при блокировке файла.
2. Проверяет, не занят ли файл через `IsFileLocked`.
3. Если файл заблокирован:
   - Показывает диалог `FileLockedDialog` с вариантами: "Повторить", "Сохранить как", "Отмена".
   - При "Сохранить как" открывает `SaveFileDialog` для выбора нового пути.
4. Удаляет существующий файл перед записью.
5. Преобразует `List<SpecificationRow>` в `List<Dictionary<string, object>>` через `ToExcelRow()`.
6. Создает структуру листов:
   - Главный лист "Спецификация" со всеми данными.
   - Дополнительные листы, сгруппированные по разделам (фильтрует строки со `Спецификация == "1"`).
7. Использует `MiniExcel.SaveAs` с конфигурацией:
   - `FastMode = true` — быстрая запись без полной загрузки в память.
   - `EnableAutoWidth = true` — автоматическая ширина столбцов.
8. Если данных нет, создает пустой файл с заголовками.

## Особенности
- Поддерживает многостраничный экспорт с группировкой по разделам.
- Обрабатывает ситуацию, когда файл Excel уже открыт пользователем.
- Использует легковесный `MiniExcel` вместо тяжелого `EPPlus` или `ClosedXML`.

## Порядок листов в Excel
1. Спецификация (все данные)
2. Детали
3. Термомосты
4. Заполнения
5. Комплектующие
6. Материалы
7. Раскрой
8. Проверка
9. Другое

## Связи с другими классами
- Вызывается из `FragmentSpecificationCollector` для финального сохранения данных.
- Использует `SpecData.BOMSections` для определения принадлежности к листу.
